# 연구 계획서(예시)
## 커머스 리뷰의 속성 기반 감성분석을 통한 제품 개선 우선순위 도출

> 본 문서는 "비즈니스 니즈를 AI 문제로 재해석"하는 과정을 보여주기 위한 예시 계획서입니다.
> 각자 선택한 프로젝트 주제와 상황에 맞게 작성해야합니다.

---

## 1. Problem Definition

### 1.1 문제점 : 비즈니스 니즈
D2C 화장품 브랜드가 신제품 리뷰 3,000여 건을 확보했다. 평균 별점은 4.1점으로 나쁘지 않지만 재구매율이 하락 중이며, CS팀은 "고객이 뭔가 불만이 있는 것 같은데 정확히 무엇인지 모르겠다"고 말한다. 요구사항은 **"무엇을(어떤 속성을) 개선해야 하는지 데이터로 제시하라"**이다.

### 1.2 Task Reformulation
- **초기 접근 방법:** "리뷰 감성분석 = 문서 단위 긍/부정 분류".
  → 그러나 이미 별점이 존재하므로 정보량이 사실상 동일하고, "발림성은 좋은데 향이 별로"처럼 한 리뷰에 상반된 감정이 섞이면 문서 단위 라벨은 의미를 잃는다. **"무엇을 고쳐야 하나"에 답하지 못한다.**
- **현재 접근 방법:** 비즈니스 질문의 핵심어는 "무엇(what)"이다. 이를 AI 문제로 옮기면 두 하위 문제로 분해된다.
  1. 리뷰에서 **평가 대상(속성) 추출** — 향, 발림성, 지속력, 가격, 용기 등 → *Information Extraction / sequence labeling*
  2. 각 속성에 대한 **감성 극성 판별** (긍정/부정/중립) → *Aspect-level Sentiment Classification*

### 1.3 Task 설계
**End-to-End Aspect-Based Sentiment Analysis (ABSA)** — 리뷰 문장에서 `(속성 term, 감성 극성)` 쌍을 모두 추출하는 문제.
- 하위 태스크: Aspect Term Extraction(ATE, IE) + Aspect Sentiment Classification(ASC)
- 최종 산출물: 속성별 부정 비율 순위표 (예: 향 62%, 지속력 48%, 발림성 20%) → **개선 우선순위**로 직결

### 1.4 이 문제가 중요한 이유
- 별점/문서 감성으로는 "무엇을" 개선할지 알 수 없다. ABSA는 이를 행동 가능한 인사이트로 바꾼다.
- 수천~수만 건 리뷰를 사람이 전수 검토하는 것은 불가능 → 자동화 가치가 크다.
- 확장성: 경쟁사 리뷰 비교, 시계열 모니터링(리뉴얼 전후), 마케팅 소구점 발굴 등으로 재사용된다.

## 2. Background & Baseline

### 2.1 Related Works
- **사전/규칙 기반:** opinion lexicon + 의존구문 분석으로 속성-의견어를 연결. 구현은 쉽지만 도메인 이식성과 재현성이 낮다.
- **Attention 기반 신경망:** 속성이 *주어졌을 때* 감성을 분류(예: ATAE-LSTM 계열). 다만 속성 추출은 별도 문제로 남는다.
- **Pipeline 방식:** ATE(개체 추출) → ASC(분류)를 각각 학습. 단순하지만 **1단계 오류가 2단계로 전파**되는 약점이 있다.
- **Pretrained LM 기반:** BERT를 활용(예: 문장+속성을 함께 입력해 분류하는 BERT-SPC, 문맥 초점을 강화한 LCF-BERT 등). 현재 강력하고 재현이 용이한 표준 baseline.
- **Unified / Generative 방식:** 하나의 seq2seq 모델이 `(속성, 감성)`(또는 의견어까지 포함한 triplet)을 직접 생성. 오류 전파가 없고 다중·중첩 속성 처리에 유리.

### 2.2 이번 프로젝트의 Baseline
- **Primary baseline: BERT 기반 Pipeline**
  1. BIO 태깅으로 속성 term 추출(ATE)
  2. 추출된 각 term에 대해 BERT-SPC 방식으로 감성 분류(ASC)
- **선정 이유:** (a) 성능이 충분히 강하고 재현이 쉬워 공정한 비교 기준이 되며, (b) pipeline 구조라 우리가 개선하려는 지점(오류 전파, 암묵적 속성 누락)이 명확하게 드러난다.
- (참고용 하한 baseline) lexicon 규칙 방식, LLM few-shot 프롬프팅 — 절대 성능 감을 잡는 용도.

## 3. Proposed Method

### 3.1 핵심 아이디어
**Generative Unified ABSA** — 인코더-디코더 모델(영어: T5 계열 / 한국어: KoBART·T5 계열)을 fine-tune하여, 리뷰가 주어지면 `속성:감성` 쌍들을 하나의 시퀀스로 직접 생성한다. 여기에 **암묵적 속성(implicit aspect) 정규화**를 결합한다. 예: "너무 비싸다"에는 '가격'이라는 표면 단어가 없지만 → `가격:부정`으로 매핑.

### 3.2 Baseline 대비 무엇이 다른가
| 구분 | Baseline (BERT pipeline) | Proposed (Generative + implicit) |
|---|---|---|
| 구조 | 2단계 분리, 오류 전파 있음 | 단일 모델, 오류 전파 없음 |
| 암묵적 속성 | 표면 span이 없으면 태깅 불가 → 누락 | 카테고리로 매핑해 포착 가능 |
| 다중/중첩 속성 | 처리 까다로움 | 시퀀스 생성으로 자연스럽게 처리 |
| 사전지식 활용 | 제한적 | LM의 사전지식으로 도메인 용어 일반화 |

### 3.3 Dataset & Preprocessing
- **사용 데이터셋**
  - (표준) SemEval-2014 Task 4 (Restaurant/Laptop), SemEval-2016 Task 5 — 라벨 품질이 높은 벤치마크.
  - (비즈니스 정합) 한국어 커머스 리뷰 — 예: AI Hub의 속성기반 감성분석 코퍼스, 또는 공개 쇼핑몰 리뷰. *(라이선스·수집방식 확인 필요)*
- **데이터 특성**
  - 라벨: (속성 term/카테고리, 극성=긍정/부정/중립)
  - 리뷰가 짧고 구어체, 오탈자·이모지·반복문자 다수
  - 클래스 불균형(중립 비율이 낮음), **암묵적 속성 표현이 상당수**
- **전처리**
  - 텍스트 정규화(반복문자 축약, 이모지·특수문자 처리), 문장 분할
  - 라벨 정합성 점검, 속성 term → 카테고리 매핑 사전 구축
  - train/dev/test 분할(도메인·극성 분포 유지), 불균형 대응(클래스 가중치 또는 샘플링)
  - Generative 학습을 위한 target 시퀀스 포맷 정의(예: `향:부정 | 지속력:부정`)

### 3.4 왜 효과가 있을 것으로 예상하는가 (검증 대상 가설)
- **H1.** 단일 생성 모델은 오류 전파를 제거하므로 pair-level F1이 pipeline보다 높을 것이다.
- **H2.** 암묵적 속성 정규화는 baseline이 놓치는 사례를 포착해 **recall**을 개선할 것이다.
- **H3.** 사전학습 LM의 지식이 저빈도 도메인 용어에 대한 일반화를 도와 소량 데이터에서 이점을 보일 것이다.

## 4. Experiment Design

### 4.1 수행할 실험
- **E1.** Baseline(BERT pipeline) 재현 및 성능 측정
- **E2.** Proposed(Generative unified) 학습 및 측정
- **E3.** Ablation — 암묵적 속성 처리 유무, target 포맷 변형의 효과 분리
- **E4.** (선택) LLM few-shot vs fine-tuned 소형 모델 비교
- **E5.** 정성 분석 — 오류 유형 분류, 특히 암묵적 속성 사례 집중

### 4.2 비교 축
- Pipeline vs Generative (주 비교)
- Explicit-only subset vs Explicit+Implicit subset 별 성능
- 학습 데이터 크기별 성능 곡선(데이터 효율)

### 4.3 Evaluation Metrics 및 검증
- **지표 설계**
  - ATE: span-level Precision/Recall/F1 (exact match)
  - ASC: Accuracy 및 **Macro-F1** (클래스 불균형 반영)
  - **End-to-end(pair): `(속성, 감성)`이 모두 맞아야 정답으로 인정 → pair-level micro/macro F1 (주 지표)**
  - 암묵적 속성 subset의 F1을 별도 보고
- **지표 검증 방법**
  - *타당성:* pair-level F1이 비즈니스 목표(속성별 개선 우선순위의 정확도)와 정렬되는지 확인
  - *신뢰성:* 3개 seed 평균 ± 표준편차 보고, 필요시 유의성 검정
  - *사람 평가:* 무작위 100건에 대해 사람 라벨과 모델 예측의 일치도(agreement) 측정
  - *비즈니스 sanity check:* 모델이 산출한 속성별 부정률 순위가 별점 낮은 리뷰의 실제 주제 분포와 부합하는지 대조
